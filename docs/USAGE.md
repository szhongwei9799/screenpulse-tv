## 1. 安装指南

### 硬件要求

| 项目 | 最低要求 | 推荐配置 |
|------|----------|----------|
| 设备 | Android TV Box / 安卓电视 | 任意 Android TV 设备 |
| Android 版本 | 7.0 (API 24) | 11.0+ (API 30+) |
| 内存 | 1 GB RAM | 2 GB+ RAM |
| 存储 | 500 MB 可用空间 | 4 GB+ |
| 网络 | Wi-Fi / 以太网 | 千兆有线网络（用于 IPTV） |
| 显示器 | 720P HDMI 显示器 | 1080P / 4K 大屏电视 |

### 安装方式

#### 方式一：ADB 远程安装（推荐）

适用于有开发者经验的用户，最快捷的方式。

```bash
# 1. 确保 Android TV 设备开启了「开发者选项」和「USB 调试」
#    大多数 Android TV Box 在「设置 → 设备偏好设置 → 关于」中
#    连续点击「版本号」7 次即可开启开发者模式

# 2. 在电脑上通过 ADB 连接设备
adb connect 192.168.1.100

# 3. 验证连接
adb devices
# 输出应显示: 192.168.1.100:5555  device

# 4. 安装 APK
adb install screenpulse-tv.apk

# 5. 如果设备上已有旧版本，使用覆盖安装
adb install -r screenpulse-tv.apk
```

#### 方式二：U 盘安装

适用于不熟悉 ADB 的普通用户。

```
1. 下载 screenpulse-tv.apk 文件
2. 将 APK 文件复制到 U 盘根目录
3. 将 U 盘插入 Android TV 设备的 USB 接口
4. 打开设备上的「文件管理器」应用
5. 找到 U 盘中的 APK 文件，点击安装
6. 如提示"未知来源"，请先在设置中允许安装未知来源应用
7. 安装完成后，在应用列表中找到「ScreenPulse TV」
```

#### 方式三：通过应用商店安装

如果上架了应用商店（如当贝市场）：

```
1. 打开 Android TV 上的应用商店
2. 搜索「ScreenPulse TV」
3. 点击安装
```

### 网络配置建议

为了获得最佳的播放体验，建议：

```
┌─────────────────────────────────────────┐
│  推荐网络拓扑                             │
│                                          │
│  [路由器]─────(有线)────[Android TV Box] │
│     │                                   │
│     │──(Wi-Fi)──[管理电脑/手机]          │
│     │                                   │
│     │──(有线/Wi-Fi)──[互联网]            │
│      (用于 IPTV/网络视频)                │
└─────────────────────────────────────────┘
```

- 📡 **Android TV 设备**建议使用**有线网络**连接路由器，确保稳定
- 📱 **管理设备**（手机/电脑）连接同一 Wi-Fi 即可
- 🌐 如需播放 IPTV 或网络流媒体，路由器需要连接互联网

## 2. 首次启动

### 启动流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  启动应用     │ ──► │  显示二维码   │ ──► │  扫码/输入IP  │
│  (电视 launcher)│    │  和管理地址   │     │  访问管理后台  │
└──────────────┘     └──────────────┘     └──────────────┘
```

### 详细步骤

1. **启动应用**
   - 在 Android TV 主界面找到「ScreenPulse TV」图标
   - 点击启动（或使用遥控器选中后按确认键）

2. **查看管理地址**
   - 首次启动后，屏幕上会显示一个**二维码**和**管理地址**
   - 地址格式为：`http://192.168.1.100:8080`
   - 屏幕同时显示设备名称和 IP 地址

3. **扫码访问**
   - 使用手机扫描屏幕上的二维码
   - 或手动在浏览器中输入显示的管理地址
   - 即可打开 Web 管理后台

4. **开始配置**
   - 按照下方「Web 管理后台使用」章节进行内容配置

## 3. Web 管理后台使用

打开 `http://{设备IP}:8080` 即可看到 Web 管理后台。

### 3.1 添加媒体文件

#### 本地上传

```
1. 在 Web 管理面板中找到「添加媒体」区域
2. 点击上传区域或拖拽文件到上传区域
3. 支持的文件格式：
   - 视频：MP4, MKV, AVI（最大 50MB）
   - 图片：JPG, PNG, GIF, WebP
4. 上传完成后自动添加到播放列表
5. 图片默认显示 10 秒，视频自动按实际时长播放
```

#### 通过 URL 添加

```
1. 在「添加网络地址」区域输入媒体 URL
2. 选择媒体类型：
   - 视频：普通网络视频文件
   - IPTV 直播：HLS/m3u8 直播流
   - 网络流：RTSP/RTMP 流媒体
   - 网页：HTTP/HTTPS 网页地址
3. 输入显示标题（可选，默认从 URL 提取）
4. 点击「添加」按钮
```

### 3.2 管理播放列表

播放列表表格支持以下操作：

| 操作 | 说明 |
|------|------|
| ✓ / ✗ | 启用/禁用该播放项（禁用后播放时跳过） |
| ↑ / ↓ | 上移/下移播放顺序 |
| ✕ | 删除该播放项 |
| 时长设置 | 可为每个项设置独立的显示时长 |
| 音量设置 | 可为每个项设置独立的音量 |

```
播放列表示例：

┌────┬──────────────────┬──────┬──────┬──────┬──────────────────┐
│ 序号│     标题         │ 类型 │ 时长 │ 音量 │     操作         │
├────┼──────────────────┼──────┼──────┼──────┼──────────────────┤
│  1 │ 产品宣传片.mp4    │video │ 自动 │ 80%  │ ✓ ↑ ↓ ✕         │
│  2 │ 门店海报.jpg      │image │ 10秒 │ 100% │ ✓ ↑ ↓ ✕         │
│  3 │ CCTV-1 直播       │iptv  │ 5分钟│ 70%  │ ✓ ↑ ↓ ✕         │
│  4 │ 公司官网          │webpg │ 30秒 │ 100% │ ✗ ↑ ↓ ✕         │
└────┴──────────────────┴──────┴──────┴──────┴──────────────────┘
```

### 3.3 设置播放参数

通过 `POST /api/settings` 或 Web 后台设置区域配置：

#### 播放模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 循环 (loop) | 播完最后一项后从第一项重新开始 | 商场促销轮播、信息展示 |
| 顺序 (sequential) | 播完最后一项后停止 | 会议演示、演出播放 |
| 随机 (random) | 每次随机选择下一项播放 | 餐厅菜单轮播、随机广告 |

#### 默认时长

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 图片显示时长 | 10 秒 | 图片在列表中未单独设置时长时的默认值 |
| 网页显示时长 | 30 秒 | 网页在列表中未单独设置时长时的默认值 |

#### 切换效果

| 效果 | 说明 |
|------|------|
| 淡入淡出 (fade) | 媒体项之间平滑过渡 |
| 无 (none) | 直接切换，无过渡动画 |

### 3.4 远程播放控制

Web 管理面板顶部的控制区域提供远程控制按钮：

| 按钮 | 功能 | 快捷键 |
|------|------|--------|
| ▶ 播放 | 开始/恢复播放 | `P` |
| ⏸ 暂停 | 暂停当前播放 | `S` |
| ⏭ 下一项 | 跳到播放列表下一项 | `→` |
| ⏮ 上一项 | 跳到播放列表上一项 | `←` |
| 🔄 刷新 | 刷新状态信息 | `R` |

### 3.5 定时插播设置

定时插播功能允许在指定时间**自动中断当前播放**，播放特定内容（如广告、通知），完成后自动恢复。

#### 使用场景

```
┌───────────────────────────────────────────────────────┐
│  日常轮播                                             │
│  ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐ ┌───────┐  │
│  │视频 A │→│图片 B │→│视频 C │→│图片 D │→│视频 A │  │
│  └───────┘ └───────┘ └───────┘ └───────┘ └───────┘  │
│                                          │            │
│                     ⏰ 12:00 触发         │            │
│                                          ▼            │
│  ┌───────────────────────────────────────────────┐    │
│  │  插播内容：午餐优惠广告 + 饮品推荐              │    │
│  │  播放完毕后自动恢复日常轮播                    │    │
│  └───────────────────────────────────────────────┘    │
└───────────────────────────────────────────────────────┘
```

#### 配置步骤

```
1. 准备插播内容（上传图片/视频或添加 URL）
2. 调用 API 创建定时任务：

   POST /api/schedule
   {
     "name": "午间广告",
     "cron": "12:00",
     "repeat": true,
     "contentJson": "[{
       \"title\": \"午餐优惠\",
       \"type\": \"image\",
       \"url\": \"/media/lunch_ad.jpg\",
       \"duration\": 15
     }]"
   }

3. 定时任务会在每天 12:00 自动触发
4. 触发时暂停当前播放，播放插播内容
5. 插播内容播完后自动恢复之前的播放位置
```

#### Cron 表达式速查

| 表达式 | 含义 |
|--------|------|
| `08:30` | 每天 8:30 |
| `1 09:00` | 每周一 9:00 |
| `5 18:00` | 每周五 18:00 |
| `1715000000` | 指定时间（Unix 秒级时间戳） |
| `09:00 18:00` | 每天 9:00 至 18:00 区间 |

## 4. 高级功能

### 4.1 IPTV 直播源配置

ScreenPulse TV 使用 ExoPlayer 播放 HLS (m3u8) 流媒体，支持大多数 IPTV 源。

#### 添加单个 IPTV 频道

```bash
# 通过 API 添加
curl -X POST http://192.168.1.100:8080/api/media/url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "http://live.example.com/cctv1/index.m3u8",
    "type": "iptv",
    "title": "CCTV-1 综合"
  }'
```

#### 支持的直播协议

| 协议 | 格式 | 说明 |
|------|------|------|
| HLS | `.m3u8` | 最常用，兼容性最好 |
| DASH | `.mpd` | 自适应流媒体 |
| RTSP | `rtsp://` | 监控摄像头常用 |
| RTMP | `rtmp://` | 直播推流协议 |

#### IPTV 注意事项

- ⚠️ 直播流没有固定时长，建议在播放列表中为 IPTV 项设置时长限制（如 300 秒 = 5 分钟），到时自动切换下一项
- ⚠️ 确保 Android TV 设备的网络带宽足够（建议 10Mbps+ 用于 1080p 直播）
- ⚠️ 部分 IPTV 源可能存在地域限制

### 4.2 网络流媒体配置

#### RTSP 监控摄像头

```bash
curl -X POST http://192.168.1.100:8080/api/media/url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "rtsp://admin:password@192.168.1.50:554/stream1",
    "type": "stream",
    "title": "门口监控"
  }'
```

#### RTMP 直播流

```bash
curl -X POST http://192.168.1.100:8080/api/media/url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "rtmp://live.example.com/live/stream_key",
    "type": "stream",
    "title": "直播间"
  }'
```

### 4.3 网页/PPT 在线展示

ScreenPulse TV 支持通过 WebView 展示在线网页，包括：

#### 支持的网页类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 公司官网 | 在线展示企业信息 | `https://www.example.com` |
| 在线文档 | Google Docs / Office Online | `https://docs.google.com/...` |
| 数据大屏 | 可视化仪表盘 | `https://grafana.example.com` |
| 公告页面 | 自建 HTML 公告页 | `http://192.168.1.200/notice.html` |
| 在线幻灯片 | PowerPoint Online | `https://1drv.ms/...` |

#### 添加网页展示

```bash
curl -X POST http://192.168.1.100:8080/api/media/url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.example.com/about",
    "type": "webpage",
    "title": "公司介绍"
  }'
```

#### 注意事项

- 网页默认显示 30 秒，可通过 `duration` 参数自定义
- WebView 启用了 JavaScript，支持交互式页面
- 需要加载的网页资源较多时，建议将时长设置长一些（如 60 秒）
- HTTPS 页面支持，混合内容也已放行

## 5. 常见问题 FAQ

### ❓ 设备相关

**Q: 如何找到设备的 IP 地址？**

A: 三种方式：
1. 启动 ScreenPulse TV 后，屏幕上会显示设备 IP
2. 在 Android TV 设置 → 网络中查看 IP 地址
3. 通过路由器管理页面查看已连接设备列表

---

**Q: 应用无法启动？**

A: 检查以下项：
1. 确保设备系统版本 ≥ Android 7.0 (API 24)
2. 确保已授予所有必要权限（网络、存储等）
3. 查看设备日志：`adb logcat -s ScreenPulseTV`

---

**Q: 开机后应用没有自动启动？**

A: 确认以下设置：
1. 在系统设置中允许 ScreenPulse TV「自启动」
2. 部分 Android TV 设备需要额外设置开机自启动权限
3. 确认已授予 `RECEIVE_BOOT_COMPLETED` 权限
4. 部分设备厂商（如华为）可能需要额外配置

### ❓ 网络相关

**Q: 浏览器无法打开管理后台？**

A: 排查步骤：
1. 确认手机/电脑与 TV 在同一局域网
2. 尝试 ping 设备 IP：`ping 192.168.1.100`
3. 检查防火墙是否阻止 8080 端口
4. 尝试关闭 VPN/代理
5. 确认 Web 服务器正在运行：`adb logcat -s WebServerManager`

---

**Q: 上传文件很慢？**

A: 建议：
1. 确保设备使用有线网络连接
2. 大文件（>20MB）建议通过 URL 方式添加
3. 可先在电脑上搭建文件服务器，再通过 URL 引用

### ❓ 播放相关

**Q: 视频播放卡顿？**

A: 可能原因和解决方案：
1. **网络带宽不足** — IPTV/网络视频需要足够带宽（1080p 建议 10Mbps+）
2. **设备性能不足** — 低端设备可能无法流畅播放 4K 视频
3. **编码格式不兼容** — 尝试将视频转码为 H.264/H.265 + AAC
4. **直播源不稳定** — 更换 IPTV 源或使用 CDN 加速的源

---

**Q: 某些 MKV 文件无法播放？**

A: MKV 是容器格式，内部编码可能不被 ExoPlayer 支持：
1. 推荐使用 H.264 (AVC) 或 H.265 (HEVC) 编码
2. 推荐使用 AAC 或 MP3 音频编码
3. 可使用 FFmpeg 转码：`ffmpeg -i input.mkv -c:v h264 -c:a aac output.mp4`

---

**Q: 图片切换时闪屏？**

A: 建议：
1. 确保所有图片分辨率接近，避免缩放闪烁
2. 启用「淡入淡出」过渡效果
3. 使用高质量图片避免加载延迟

### ❓ 定时任务相关

**Q: 定时任务没有触发？**

A: 检查以下项：
1. 确认设备没有被电池优化（关闭 Doze 模式）
2. 确认已授予 `SCHEDULE_EXACT_ALARM` 权限
3. 检查 cron 表达式格式是否正确
4. 查看日志：`adb logcat -s ScheduleManager`
5. 部分 Android 版本对后台定时有限制
