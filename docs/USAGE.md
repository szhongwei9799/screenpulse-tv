## 安装指南

### 系统要求

| 项目 | 最低要求 |
|------|---------|
| Android 版本 | 5.0 (Lollipop) / API 21 及以上 |
| 设备类型 | Android TV 机顶盒、电视棒、智能电视 |
| 网络 | Wi-Fi 或有线网络（用于 Web 后台管理） |
| 存储 | 至少 100MB 可用空间（用于应用和上传文件） |

### 安装方式

#### 方式一：ADB 安装（推荐开发者使用）

1. 在 Android TV 上开启 **开发者选项** 和 **USB 调试**
   - 进入 设置 > 关于 > 连续点击"版本号" 7 次开启开发者模式
   - 进入 设置 > 开发者选项 > 开启 USB 调试
2. 用 USB 线连接电脑和 Android TV
3. 在电脑上执行：

```bash
adb install screenpulse-tv-player.apk
```

#### 方式二：U 盘安装

1. 将 APK 文件复制到 U 盘根目录
2. 将 U 盘插入 Android TV 的 USB 接口
3. 在 TV 上打开文件管理器（如 ES 文件浏览器）
4. 找到 U 盘中的 APK 文件，点击安装
5. 如果提示"未知来源"，请前往 设置 > 安全 > 允许未知来源安装

#### 方式三：通过应用商店

如果已上架到应用商店（如当贝市场、小米应用商店等），可直接搜索 **ScreenPulse** 下载安装。

### 安装后首次设置

安装完成后，建议执行以下操作：

1. **授予存储权限** — 允许应用访问本地媒体文件
2. **设置为开机自启** — 确保"开机自动启动"已开启（应用会自动注册）
3. **连接 Wi-Fi** — 确保设备和手机/电脑在同一局域网内

## 首次启动

### 启动画面

安装完成后，在 Android TV 主界面找到 **ScreenPulse** 应用图标，点击启动。

### 二维码和管理地址

首次启动时，屏幕中央会显示一个 **QR 二维码** 和管理地址，类似如下：

```
┌──────────────────────────────────────────┐
│                                          │
│          ┌──────────────┐               │
│          │              │               │
│          │   QR Code    │               │
│          │              │               │
│          └──────────────┘               │
│                                          │
│    管理地址：http://192.168.1.105:8080    │
│                                          │
│    使用手机扫描二维码或输入上方地址        │
│    即可在浏览器中管理播放列表              │
│                                          │
└──────────────────────────────────────────┘
```

### 如何连接

1. 确保手机/电脑与 Android TV 连接在 **同一个 Wi-Fi 网络**
2. **方式 A**：用手机扫码 App 上的二维码
3. **方式 B**：在浏览器地址栏手动输入显示的管理地址（如 `http://192.168.1.105:8080`）
4. 浏览器将打开 Web 后台管理界面

### 播放列表为空

当播放列表中没有媒体内容时，屏幕会持续显示二维码和管理地址，提醒您添加内容。添加媒体后，播放将自动开始。

## Web 后台管理

打开管理地址后，您将看到 Web 管理界面。界面采用暗色主题设计，包含以下主要模块：

### 仪表盘 (Dashboard)

仪表盘显示当前设备的运行状态：

| 信息项 | 说明 |
|--------|------|
| 设备名称 | 当前 Android TV 的设备型号 |
| IP 地址 | 设备在局域网中的 IP 地址 |
| 管理地址 | 完整的 Web 管理访问 URL |
| 活动项目数 | 当前已启用的播放列表项目数量 |
| 播放模式 | 当前播放模式（循环/顺序/随机） |
| 插播状态 | 是否启用了定时插播 |
| 音量级别 | 当前音量设置 (0-100) |
| Android 版本 | 设备系统版本 |
| 应用版本 | ScreenPulse TV Player 版本号 |

**操作按钮：**
- **刷新状态** — 手动刷新设备信息
- **扫描本地文件** — 自动扫描设备上的媒体文件

### 播放列表管理

这是最常用的功能模块，用于管理所有播放内容。

#### 添加媒体项

1. 点击页面右上角的 **"添加"** 按钮
2. 在弹出对话框中填写：
   - **标题** — 媒体名称（必填）
   - **URL/路径** — 本地文件路径或网络地址（必填）
   - **媒体类型** — 从下拉菜单选择（视频/图片/PPT/IPTV/流媒体）
   - **播放时长** — 图片/PPT 的显示时间（秒），视频可不填（自动检测）
   - **启用** — 是否加入播放循环
3. 点击 **"确认"** 保存

#### 编辑媒体项

1. 在播放列表中找到要修改的项
2. 点击该项右侧的 **编辑按钮**（铅笔图标）
3. 修改相关字段
4. 点击 **"确认"** 保存

> 修改会实时生效，正在播放的内容会在下一个切换周期更新。

#### 删除媒体项

1. 点击该项右侧的 **删除按钮**（垃圾桶图标）
2. 确认删除

> 如果删除的是通过后台上传的本地文件，文件也会从设备存储中一并删除。

#### 拖拽排序

1. 按住媒体项左侧的 **拖拽手柄**
2. 上下拖动到目标位置
3. 松开鼠标/手指

排序变更会立即生效。您也可以使用批量排序 API 一次性调整多个项目的顺序。

#### 启用/禁用

每个媒体项旁边有一个 **开关按钮**：
- **开启（绿色）** — 该项参与播放循环
- **关闭（灰色）** — 该项暂停播放，但保留在列表中

> 禁用不等同于删除。您可以随时重新启用被禁用的项目。

### 媒体库管理

用于上传和管理媒体文件。

#### 上传文件

1. 进入 **媒体库** 页面
2. 点击 **上传区域** 或直接 **拖拽文件** 到上传区域
3. 支持 **批量上传** 多个文件
4. 勾选 **"自动添加到播放列表"** 可在上传后自动创建播放列表项
5. 等待上传完成

**支持的文件格式：**
- 视频：MP4、AVI、MKV、MOV、3GP、WebM、FLV
- 图片：JPG、JPEG、PNG、GIF、BMP、WebP
- 演示文稿：PPT、PPTX、PDF
- 流媒体：M3U、M3U8

#### 扫描本地文件

1. 点击 **"扫描本地文件"** 按钮
2. 系统将自动扫描以下目录：
   - `/存储/ScreenPulse/`
   - `/存储/DCIM/`（相机照片）
   - `/存储/Movies/`（视频文件）
   - `/存储/Pictures/`（图片文件）
   - 应用内部上传目录
3. 新发现的文件会自动添加到播放列表（不会重复添加已有文件）

### 设置

用于配置全局播放参数。

#### 播放模式

| 模式 | 说明 |
|------|------|
| **循环播放** | 播放完最后一个项目后，自动从第一个重新开始，永不停止 |
| **顺序播放** | 按列表顺序播放一遍，播放完毕后停止，显示二维码 |
| **随机播放** | 每次随机选取下一个项目，确保不会连续播放同一个 |

#### 定时插播

定时插播功能允许在指定时间段内切换为插播内容（例如午休时段播放广告）。

1. **启用插播** — 打开定时插播开关
2. **开始时间** — 设置插播开始的小时（0-23），例如 12 表示 12:00
3. **结束时间** — 设置插播结束的小时（0-23），例如 13 表示 13:00
4. 点击 **保存**

> 支持跨午夜设置（如 22:00 到 06:00），系统会自动处理时间跨越逻辑。

#### 音量控制

- 使用滑块调节音量（0 = 静音，100 = 最大音量）
- 默认音量为 80%
- 调整后立即生效

## 播放列表功能说明

### 媒体类型支持

ScreenPulse TV Player 支持多种媒体类型的全屏轮播播放：

#### 视频

| 格式 | 说明 |
|------|------|
| MP4 | 最常用的视频格式，推荐使用 H.264 编码 |
| AVI | 常见 PC 视频格式 |
| MKV | 支持多音轨/多字幕的容器格式 |
| MOV | Apple QuickTime 格式 |
| 3GP | 3G 移动设备格式 |
| WebM | 开源 Web 视频格式 |
| FLV | Adobe Flash 视频格式 |

> 视频播放结束后自动切换到下一项。时长由视频本身决定，无需手动设置。

#### 图片

| 格式 | 说明 |
|------|------|
| JPG / JPEG | 最常用的照片格式 |
| PNG | 支持 PNG 透明通道 |
| GIF | 支持 GIF 动画 |
| BMP | Windows 位图格式 |
| WebP | Google 高效图片格式 |

> 图片默认显示 **10 秒**，可在添加时自定义播放时长。

#### 演示文稿 (PPT)

| 格式 | 说明 |
|------|------|
| PPT / PPTX | Microsoft PowerPoint 演示文件 |
| PDF | PDF 文档 |

> PPT/PDF 默认显示 **60 秒**，可在添加时自定义播放时长。

#### IPTV 直播流

| 格式 | 说明 |
|------|------|
| M3U / M3U8 | HLS 直播流播放列表 |

> IPTV 为持续播放的直播流，不会自动切换到下一项。如需切换，请在 Web 后台手动操作。

#### 网络流媒体

| 协议 | 说明 |
|------|------|
| HTTP / HTTPS | 直接播放网络上的视频文件 |
| HLS (m3u8) | HLS 自适应码率流 |
| DASH (mpd) | MPEG-DASH 自适应流 |
| RTSP | 实时流传输协议 |

### 播放时长设置

不同媒体类型的时长设置规则：

| 媒体类型 | 时长设置 | 默认值 |
|----------|---------|--------|
| 视频 | 自动检测（不可自定义） | 由视频文件决定 |
| 图片 | 可自定义（秒） | 10 秒 |
| PPT/PDF | 可自定义（秒） | 60 秒 |
| IPTV 直播流 | 持续播放（不可自定义） | 一直播放 |
| 网络流媒体 | 持续播放（不可自定义） | 一直播放 |

### 播放模式

#### 循环播放（推荐）

- 列表中的所有 **已启用** 项目按顺序播放
- 播放完最后一个后自动从第一个重新开始
- 适合长期展示场景（如门店显示屏）

#### 顺序播放

- 按列表顺序播放一遍
- 播放完毕后停止，显示二维码和管理地址
- 适合一次性展示场景（如会议等候区）

#### 随机播放

- 每次随机选取下一个项目
- 保证不会连续播放同一个项目
- 适合广告轮播场景，增加新鲜感

### 定时插播

定时插播是 ScreenPulse 的特色功能，允许您在特定时间段插入专属内容。

#### 使用场景举例

| 场景 | 时间段 | 正常内容 | 插播内容 |
|------|--------|---------|---------|
| 餐厅午休广告 | 12:00–13:00 | 菜单介绍 | 促销广告 |
| 商场晚间促销 | 18:00–21:00 | 品牌形象 | 折扣活动 |
| 办公楼午间 | 12:00–13:30 | 企业文化 | 员工福利通知 |
| 24小时滚动 | 22:00–06:00 | 白天内容 | 夜间特别节目 |

#### 配置步骤

1. 在 Web 后台进入 **设置** 页面
2. 打开 **定时插播** 开关
3. 设置 **开始时间** 和 **结束时间**（以小时为单位，0-23）
4. 点击 **保存**

#### 工作原理

- 系统每 15 分钟检查一次当前时间
- 当时间进入插播窗口时，自动切换到插播内容
- 当时间离开插播窗口时，自动恢复正常播放列表
- 如果需要立即生效，可重启应用

## 网络资源播放

ScreenPulse TV Player 支持直接播放网络上的媒体资源，无需提前下载。

### 支持的协议

| 协议 | 格式 | 使用场景 |
|------|------|---------|
| **HTTP/HTTPS** | MP4, WebM 等 | 播放在线视频文件 |
| **HLS** | M3U8 | IPTV 直播流、在线直播 |
| **DASH** | MPD | 自适应码率流媒体 |
| **RTSP** | RTSP URL | 安防摄像头、IP 摄像头实时画面 |

### 添加网络资源

1. 进入 Web 后台 → **播放列表**
2. 点击 **添加**
3. 在 URL/路径栏输入网络地址，例如：
   - HLS 直播流：`https://example.com/live/stream.m3u8`
   - 在线视频：`https://example.com/videos/promo.mp4`
   - RTSP 摄像头：`rtsp://192.168.1.200:554/live`
4. 选择正确的媒体类型
5. 点击 **确认** 保存

### 注意事项

- 网络播放需要 Android TV 设备有稳定的 **网络连接**
- HLS/DASH 流会根据网络状况 **自动调整画质**
- RTSP 播放依赖设备上安装的 ExoPlayer 解码能力
- 如果网络地址无法访问，该项会 **自动跳过** 并播放下一项
- 建议在添加前先在电脑上测试网络地址是否可访问

## 常见问题 FAQ

### Q: 扫描二维码后无法打开管理页面？

**A:** 请检查以下几点：
1. 手机/电脑是否与 Android TV 连接在 **同一个 Wi-Fi 网络**
2. TV 是否已正确连接网络（可以在 TV 设置中查看 IP 地址）
3. 是否有防火墙拦截了 8080 端口
4. 尝试直接在浏览器中输入 IP 地址和端口（如 `http://192.168.1.105:8080`）

### Q: 上传文件失败？

**A:** 可能原因：
1. 文件大小超过限制（建议单个文件不超过 500MB）
2. 磁盘空间不足（请清理 TV 存储）
3. 网络不稳定（如通过 Wi-Fi 上传大文件）
4. 文件名包含特殊字符（建议使用英文和数字命名）

### Q: 视频播放卡顿？

**A:** 优化建议：
1. 确保视频编码为 **H.264**，分辨率不超过 1080p
2. 检查 TV 的存储读取速度（建议使用高速 SD 卡或内置存储）
3. 如果播放网络视频，请确保网络带宽充足
4. 尝试降低视频比特率或分辨率

### Q: 播放列表修改后没有立即生效？

**A:** 播放列表的修改会在当前播放项结束后自动生效。如需立即生效，可以：
1. 在 Web 后台点击"刷新"
2. 或在 TV 上重启应用

### Q: 设备重启后应用没有自动启动？

**A:** 请检查：
1. 应用是否已获得"开机自启动"权限
2. 部分厂商（如小米、华为）需要在系统设置中手动允许应用开机自启
3. 确认 BootReceiver 已正确注册（通常不需要手动操作）

### Q: 屏幕一直显示二维码？

**A:** 这表示播放列表为空。请通过 Web 后台添加至少一个媒体项。添加后播放会自动开始。

### Q: 如何同时管理多台设备？

**A:** 目前每台设备需要独立管理。每台设备有独立的 IP 地址和 Web 管理界面。未来版本计划支持集中管理功能。

### Q: 支持竖屏播放吗？

**A:** 应用默认以横屏模式运行（`android:screenOrientation="landscape"`），适配电视和显示器。暂不支持竖屏模式。

### Q: 如何备份播放列表？

**A:** 您可以通过 API 导出播放列表数据：
```bash
curl http://设备IP:8080/api/playlist > playlist_backup.json
```

## 键盘快捷键

如果您连接了蓝牙键盘或使用 Android TV 遥控器，可以使用以下快捷操作：

| 按键 | 功能 |
|------|------|
| `Enter` / OK | 确认/播放 |
| `Back` / Esc | 返回/暂停 |
| `←` | 上一个媒体项 |
| `→` | 下一个媒体项 |
| `Space` | 暂停/继续 |
| `↑` | 音量增加 |
| `↓` | 音量减少 |
| `M` | 静音/取消静音 |
| `F` | 全屏切换（已默认全屏） |
| `R` | 刷新当前播放项 |
| `Q` | 显示/隐藏二维码 |

> 注意：部分按键可能因遥控器型号不同而有所差异。
