# ScreenPulse TV — 设计需求规格文档

> **重要说明**：本文档是项目的唯一设计需求基准。
> - 任何新增/修改的需求必须如实追加到本文档中。
> - **每次构建前必须逐项对比本文档，确认每一项需求均已满足。**
> - 修改需求时需注明修改日期和修改原因。

---

## 一、整体架构

| # | 需求 | 优先级 | 状态 |
|---|------|--------|------|
| A01 | Android TV 数字标牌轮播播放器 | P0 | ✅ 已实现 |
| A02 | NanoHTTPD 内嵌 Web 服务器，端口 8080 | P0 | ✅ 已实现 |
| A03 | Web 管理后台：Vue 3 + Element Plus 单文件 SPA（`index.html`） | P0 | ✅ 已实现 |
| A04 | 前端从 Android assets 目录（`app/src/main/assets/web-admin/`）直接提供，无需额外构建 | P0 | ✅ 已实现 |
| A05 | 项目主模块为 `app/`，包名 `com.screenpulse.tv` | P0 | ✅ 已实现 |
| A06 | Room 数据库存储媒体、播放列表、分组、背景音乐等 | P0 | ✅ 已实现 |
| A07 | GitHub Actions 自动构建 APK 并创建 Release | P1 | ✅ 已实现 |
| A08 | 全中文界面（zh-CN） | P0 | ✅ 已实现 |
